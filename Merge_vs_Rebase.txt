what is diff between git merge and git rebase?

Merge - зливання/сливание - ствоюється новий комміт, але старі залишаються в іншій гілці. 
Коли багато людей проацює в одній гілці та потрібна повна історія комітів, то мердж для цього підійде
Rebase - перебазування - перезаписується історія комітів, створюєтося єдина лінія/лєсєнка комітів
